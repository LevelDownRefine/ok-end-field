# 装备词条识别轮子

旧的“毕业基质识别”一次性任务已移除，不再提供自动遍历、上锁、解锁或弃置等 UI 自动化行为。

本目录仅保留可复用的识别轮子，供后续任务或离线工具按需组合。

---

## 保留内容

### OCR 面板解析

位置：`src/essence/essence_recognizer.py`

能力：

- `parse_essence_panel(texts)`：从 OCR Box 列表解析面板名称、来源、词条和等级。
- `read_essence_info(task)`：通过任务上下文截图 + OCR 后调用 parser，作为后续任务的便捷入口。
- `attach_essence_levels(info, level_boxes)`：将单独 OCR 到的等级 Box 按垂直顺序附加到已解析词条。

### CSV 规则匹配

位置：`src/essence/weapon_data.py`

能力：

- `load_weapon_data(path)`：读取 `weapon_data*.csv`。
- `discover_weapon_data_language_options(dir)`：发现多语言 CSV 选项。
- `match_weapon_requirements(requirements, entry_names)`：按词条集合匹配规则。

### 模板资产

位置：`assets/` 与 `ok_templates/`

保留 `essence_*` 模板和 COCO 类别，用于后续重新组合 UI 检测流程；当前项目不再注册对应任务。

## 后续复用建议

如果要重新实现相关能力，建议分层组合：

1. 任务层只负责进入页面、选择格子、滚动等 UI 流程。
2. OCR 层调用 `read_essence_info(task)` 或 `parse_essence_panel(texts)`。
3. 规则层调用 `load_weapon_data()` 与 `match_weapon_requirements()`。
4. 危险操作（上锁、解锁、弃置）必须单独加确认配置和日志，不应写入解析轮子。

这样可以让解析能力复用于测试、离线分析、调试任务或未来的新 UI 流程。
